# Phase 4: Interactive Manifest Builder

## Overview

Implement `scripts/build_manifest.py` + fill `references/workflow-pack.md` (question bank + LLM orchestration prose). Drives the **interactive fallback** path: when PO invokes `/cleanmatic:claude-pack` with no manifest and no CLI category flags, the LLM walks them through category selection via `AskUserQuestion` and writes a fresh `.claude/pack.manifest.yaml`.

**Critical architectural rule (Researcher B audit + LOCKED API Contracts):** Python scripts NEVER call `AskUserQuestion`. The LLM is the UI layer; the script is the structural layer. `build_manifest.py` has three modes:
- `--discover` — scan `.claude/` to enumerate available items.
- `--list-questions` — emit question bank as JSON (LLM uses for AskUserQuestion).
- `--write` — accept answers JSON on stdin, validate via `manifest_loader.validate`, atomic-write YAML.

Mirrors product-spec's `workflow-interview.md` pattern.

## Requirements

### build_manifest.py

**Functional:**
- `--discover --root <project>` — enumerate disk state; emit JSON `{available_skills, available_agents, available_hooks, available_rules, has_readme, has_claudemd, has_settings, has_ck_config}`.
- `--list-questions [--lang en] [--root <project>]` — internally calls `--discover`; emit JSON `{batches: [{id, questions: [{id, header, question, type, options|default}]}]}`. Covers 4 batches per workflow-pack.md.
- `--write [--out .claude/pack.manifest.yaml] [--force] [--root <project>]` — read stdin → JSON → assemble manifest dict → validate via `manifest_loader.validate(manifest, root)`; if errors: stderr + exit 1. **Atomic write** (validate F1.4): write to `out.with_suffix(out.suffix + ".tmp")`, fsync, `os.replace`. Refuse if `out.exists() and not force` → exit 2.
- All output JSON to stdout; errors to stderr.
- **Exit codes:** 0 success · 1 validation error · 2 file-exists collision without `--force`.

**Canonical YAML key order (LOCKED per validate F3.4):**
```
schema_version, version, bundle_name, skills, agents, hooks, rules, extra, top_level, defaults, follow_shared
```
Single source-of-truth; written here in this phase + used by tests.

**YAML header comment (LOCKED):**
```yaml
# Generated by cleanmatic:claude-pack build_manifest.py
# Edit freely; this is the source-of-truth for pack/cli.py.
# Schema: .claude/skills/claude-pack/references/manifest-spec.md
```

**Non-functional:**
- Idempotent: identical answers → byte-identical YAML.
- Output YAML: `yaml.safe_dump(manifest, default_flow_style=False, sort_keys=False)` with explicit key reordering; 2-space indent.
- pyyaml pinned `>=6.0,<7` in Phase 1's `requirements.txt`.

### references/workflow-pack.md (fill from Phase 1 stub)

**5 sections:**
1. **When this flow runs** — no-flag invocation + no existing manifest (or `--interactive` + existing-manifest overwrite-prompt).
2. **Question Bank** — 4 batches with question IDs + prose + expected types. EN only for v1 (R3 + validate scope).
3. **LLM Orchestration Pattern** — pseudocode:
   ```
   bash: build_manifest.py --list-questions --discover --root .  → questions JSON
   for batch in questions.batches:
       answers[batch.id] = AskUserQuestion(batch.questions)  ← LLM tool call
   summary = render_summary(answers)
   confirmed = AskUserQuestion("Ready to write .claude/pack.manifest.yaml with {N} skills, {M} agents? [y/N]")
   if confirmed: bash: echo $answers_json | build_manifest.py --write --root .
   ```
4. **Validation hook** — LLM must show confirmation summary AND require explicit user confirm before `--write`.
5. **Resume path** — abort mid-interview: LLM's context holds prior answers; LLM can resume with "I have your answers from batches 1-3; ready for batch 4?". No disk persistence (KISS; product-spec uses `.session.md` because batches are larger).

**Confirm prompt (LOCKED per validate F10.4):**
> `"Ready to write .claude/pack.manifest.yaml with {N} skills, {M} agents, {K} rules? [y/N]"`

## Implementation Steps

1. **Constants** — `CANONICAL_KEY_ORDER`, `HEADER_COMMENT`, `DEFAULTS` (mirror `manifest_loader.DEFAULTS` to avoid drift — import directly if possible to enforce single source-of-truth).

2. **`discover(root)`** — enumerate:
   - `available_skills`: list `.claude/skills/` subdirs containing `SKILL.md` (exclude `_shared/`, `.venv/`)
   - `available_agents`: list `.claude/agents/**/*.md` basenames stripped (recursive — supports nested per red-team F17.1)
   - `available_hooks`: list `.claude/hooks/*` non-dir entries with extensions preserved
   - `available_rules`: list `.claude/rules/**/*.md` basenames stripped (recursive)
   - `has_*` booleans for top-level files

3. **`list_questions(discovery, lang)`** — assemble 4 batches:
   - **Batch 1 (version-and-name):** 2 questions
   - **Batch 2 (categories):** 4 questions (skills, agents, hooks, rules — multi-select, options from discovery)
   - **Batch 3 (top-level):** multi-select over 4 booleans
   - **Batch 4 (extra + follow-shared):** free-text for extra (with note "no absolute paths, no `..`"); boolean for follow_shared with explanation it's warn-only
   - Localization placeholder for `lang == "vi"` (skip in v1; emit English only).

4. **`write_manifest(answers, out_path, root, force)`** —
   - Build manifest dict from answers, ordering keys per `CANONICAL_KEY_ORDER`
   - `errors = manifest_loader.validate(manifest, root)`; if errors → write errors to stderr; exit 1
   - If `out_path.exists() and not force` → exit 2
   - Render via `yaml.safe_dump` + reorder keys (post-process: rebuild dict in order); prepend `HEADER_COMMENT`
   - `_atomic_write_yaml(content, out_path)`: write to `tmp_path = out_path.parent / f".{out_path.name}.tmp"`; fsync; `os.replace(tmp_path, out_path)`
   - Print confirmation JSON `{ok: true, path: str(out_path), bytes: N}` to stdout
   - Exit 0

5. **`_atomic_write_yaml(content, out_path)`** — exactly:
   ```python
   tmp = out_path.parent / f".{out_path.name}.tmp"
   try:
       with open(tmp, "w", encoding="utf-8", newline="\n") as f:
           f.write(content)
           f.flush()
           os.fsync(f.fileno())
       os.replace(tmp, out_path)
   except Exception:
       tmp.unlink(missing_ok=True)
       raise
   ```

6. **CLI `main()`** — `argparse` with mutually-exclusive `--discover | --list-questions | --write`. Common `--root` (default CWD), `--lang en`, `--out`, `--force`. For `--write`, `sys.stdin.read()` → `json.loads` → answers dict.

7. **Fill `workflow-pack.md`** — 5 sections per Requirements. Pin confirm prompt verbatim. Reference manifest-spec.md + flag-reference.md.

8. **Cross-check `SKILL.md`** — confirm no-flag menu in SKILL.md Step 4 (from Phase 1) actually points to this workflow. If not, 5-line edit to align.

9. **Per-phase compile** (validate F15.3): `python -m py_compile scripts/build_manifest.py` exits 0.

## Risk Assessment

- **R1: Question bank desynced from manifest schema** → Phase 5 test: derive expected question IDs from `manifest_loader.DEFAULTS` keys; assert `list_questions` covers all.
- **R2: LLM walks batches but skips confirmation** → `workflow-pack.md` explicitly documents the confirm step + the locked prompt text. SKILL.md no-flag menu repeats the rule.
- **R3: User edits manifest by hand, then runs interactive again** → `--write` exits 2 by default if file exists. LLM workflow MUST prompt "overwrite or read existing?" before invoking `--write --force`.
- **R4: Discovery slow on large repos** → `os.scandir` (non-recursive at top, recursive per agents/rules). Acceptable for ≤200 skills (validate F3.6 weasel-word fix: explicit upper bound).
- **R5: YAML serialization non-deterministic across pyyaml versions** → `pyyaml>=6.0,<7` pinned (plan Cross-Phase Mutations); `sort_keys=False` + explicit reordering; Phase 5 test asserts byte-identity.
- **R6: Atomic-rename race during `--write`** → atomic-write recipe handles via `os.fsync` + `os.replace`. POSIX + Windows both atomic.

## Security Considerations

- Answers JSON from stdin: treat as untrusted. Validate via `manifest_loader.validate` (Phase 3) before write.
- File-write target restricted to within repo root: validate `out_path.resolve().is_relative_to(root.resolve())`. Reject if outside.
- Never `eval()` or `yaml.load()` (unsafe). Only `yaml.safe_load` / `yaml.safe_dump`.
- HEADER_COMMENT clearly marks file as generated; reviewers know the origin.
